# _backup 目录

此目录用于存放开发过程中的临时文件和备份。

## 用途

- 旧版本文档备份
- 临时导出的数据
- 废弃但可能有参考价值的代码
- 会话日志和调试信息

## 注意事项

**请定期清理此目录！**

建议每周或每个迭代结束时：
1. 删除不再需要的备份文件
2. 将有价值的内容整理到正式文档中
3. 保持目录整洁，避免积累过多历史文件

## 建议的子目录结构

```
_backup/
├── legacy_docs/      # 旧版文档
├── legacy_code/      # 废弃代码片段
├── session_logs/     # AI 会话日志
└── temp/             # 临时文件
```

---

_此目录不应包含敏感信息，且应定期清理_
